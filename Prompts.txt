I am reviewing the layout for the Apollo Quant Trading application and there is a structural responsiveness issue across both mobile and desktop views.

On mobile, the feature cards are constrained to the left side, while the disclaimer text is floating out of place on the right.
On desktop, there is an unintended side by side split. The main content is trapped in the left half of the screen, while the disclaimer text is floating in an empty right half.

Please update the component to fix these views. Make the following structural adjustments:

Vertical Flow: The main page wrapper must be a vertical stack on all screen sizes. Remove any responsive row based flex properties or multi column grid classes that are forcing the side by side layout on larger screens.

Desktop Centering: Center the main content container horizontally on the screen. Use auto margins along with a reasonable maximum width to ensure the app sits centered in the middle of the desktop viewport.

Mobile Scaling: The main wrapper must take up the full width on mobile screens instead of retaining a fixed desktop width. Force the layout to stack vertically on small breakpoints and only expand the inner cards to multiple columns on medium or large screens.

Footer Repositioning: Move the disclaimer text and links to the bottom of the normal vertical document flow. It should sit completely below the main application content and be centered horizontally on all devices.
